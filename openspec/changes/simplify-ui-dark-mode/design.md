## Context

現状のUIはヘッダーや補助情報が多く、コンテナ操作に集中しづらい。SvelteフロントエンドとTauriのウインド制御があり、OS設定に追従するダークモードと、コンテナ数に応じたウインドサイズ調整が必要。仕様は `proposal.md` に定義済み。

## Goals / Non-Goals

**Goals:**
- コンテナ名 + 操作ボタンのみの最小UIを実現する
- OS設定に追従したライト/ダークの自動切り替えを追加する
- 初期表示と増加時のみウインドサイズを拡張し、減少時は維持する

**Non-Goals:**
- 新しいコンテナ操作機能の追加
- ユーザーが切り替えるテーマUIの追加
- 既存のコンテナAPIの変更

## Decisions

- **UI構成の最小化**: ヘッダー、詳細情報、装飾要素を削除し、リストは「名前 + ボタン」だけに限定する。認知負荷を減らすために情報を最小化する方針を採用。
- **ダークモードの実装**: CSSで `prefers-color-scheme` を利用し、OS設定に応じた配色に自動切り替えする。アプリ内切り替えは要件外。
- **ウインドサイズ拡張ポリシー**: 初期表示時にコンテナ数から推定サイズを算出して設定し、リフレッシュ後は前回最大数を記録して増加時のみ拡張する。減少時は縮小しないことでUXの揺れを防ぐ。
- **制御箇所**: サイズ算出と拡張判定はフロントで行い、Tauri API経由でウインドサイズを変更する。既存のフロント/バックエンド構成に沿って実装負担を最小化する。

## Risks / Trade-offs

- **ウインドサイズ拡張の計測誤差** → 実測に基づく行高・余白の定数化で調整し、実機確認で微修正する
- **OSテーマ判定の差異** → `prefers-color-scheme` に依存し、OS/ブラウザ差異がある環境は既定のライト配色にフォールバックする
- **コンテナ数が急増するケース** → 上限サイズを設け、一定以上はスクロールで対応する
