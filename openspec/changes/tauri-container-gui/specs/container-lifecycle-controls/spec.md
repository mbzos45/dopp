## Overview

コンテナの起動/停止/再起動をGUIから実行するための操作仕様を定義する。

## Operations

- `start`: 停止中のコンテナを起動する。
- `stop`: 稼働中のコンテナを停止する。
- `restart`: 稼働中のコンテナを再起動する。

## Behavior

- すべての操作はコンテナIDで指定する。
- `stop` と `restart` はエンジンのデフォルトタイムアウトに従う。
- 操作成功時は、UI側で状態再取得を行い最新の状態を反映する。

## Error Handling

- 操作失敗時は、UIに操作名付きの短いメッセージを表示する。
	- 例: "コンテナの停止に失敗しました。"
- 失敗理由はログに残す。
